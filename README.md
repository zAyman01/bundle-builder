# Security System Bundle Builder

A responsive, data-driven bundle builder built for Wyze. Step-through accordion on the left, live review panel on the right.

## Quick Start

```bash
npm install
npm run dev          # http://localhost:5173
```

Or with the optional backend API:

```bash
npm install
npm run dev:all      # API on :3001 + Vite on :5173
```

Build:

```bash
npm run build        # → dist/
```

Requires **Node 18+**.

## Stack

- **React 19 + TypeScript + Vite** — modern, type-safe foundation
- **Tailwind CSS v4** — utility-first, no config file
- **Single hook** (`useBundleState`) — all state, derived computations, and localStorage persistence in one place
- **Express (bonus)** — `server/index.js` serves `GET /api/bundle`; client auto-falls back to static data if API is unavailable

## Architecture

```
server/
  index.js              Express server (bonus)
  data.json             Bundle data served via API
src/
  data/bundle.ts        Static fallback data (same shape as API response)
  types/index.ts        Product, Variant, PlanOption, Step types
  hooks/useBundleState.ts     All state, memoized derivations, save/restore
  components/
    StepAccordion.tsx   4-step accordion with product grid / plan cards
    ProductCard.tsx     Badge, image, description, variant chips, stepper, pricing
    QuantityStepper.tsx Shared ± control (card and review line)
    ReviewPanel.tsx     "Your security system" — items, plan, shipping, totals
    icons.tsx           SVG icons + PNG step icon wrappers
```

Every card and review line is generated by mapping over `steps` / `reviewLines`. Adding a product = adding a JSON entry; no component changes needed.

## Key Interactions

| Feature | Implementation |
|---|---|
| **Variant-aware quantities** | Each `productId:variantId` pair has its own count. The card stepper shows the active variant's count; switching variants never touches other counts. The review panel lists every variant > 0 separately. |
| **Synced steppers** | Card and review steppers read/write the same state slice — changes in either place update both instantly, along with the total. |
| **Accordion** | Step 1 open on load. `grid-template-rows: 0fr ↔ 1fr` animation for smooth, measured open/close. |
| **"N selected"** | Counts distinct products with any variant > 0 in that step. |
| **Live total** | Derived via `useMemo` chain: reviewLines → oneTimeTotal → grandTotal → savings/financing. Recalculates on every quantity/plan change. |
| **Save for later** | Explicit button → snapshots quantities/variants/plan to `localStorage`. Page reload restores exact state. |
| **Checkout** | Processing spinner (1.5s) → "Thanks for your order!" (3s) → auto-reset to empty state. |
| **Empty state** | Show "No items selected yet" when nothing configured. Pricing/checkout hidden until you add items. |

## Responsive Breakpoints

| Viewport | Cards/row | Review panel |
|---|---|---|
| `< 640px` (phone) | 1 | Single column |
| `640–768px` (small tablet) | 2 | Single column |
| `768–1024px` (tablet) | 3 | 2-column split |
| `1024–1280px` (desktop) | Grid of 5 | Single column |
| `1280px+` (wide) | 2 (flex, centered) | Sticky right column |

Page stacks to single column below `xl` (1280px). Product grid uses flex-wrap with `justify-center` so partial last rows are naturally centered.

## Design Decisions

- **No external state library.** A single hook with `useState` + `useMemo` + `useCallback` handles everything — simpler to reason about than Redux/Zustand for this scope.
- **`allProducts` flattening is a cache of `steps`.** Keeps `getProductById` O(1) lookups without repeated flatMap. Re-memoized when `steps` updates from the API.
- **Pricing is always computed, never mocked.** Unit price × quantity for every line. Grand total = one-time sum + selected plan. This is internally consistent even if it diverges from static mocks.
- **API-first with transparent fallback.** The client tries `GET /api/bundle` on mount. If the server is down, it falls back to the static TypeScript module — no build-time config, no environment variables.
- **`invisible` on collapsed accordion content.** Keeps content in the DOM for smooth CSS transitions but removes it from the accessibility tree and tab order so collapsed steps can't trap keyboard focus.
- **No zoom lock.** Viewport meta allows user zoom, per WCAG.

## Known Gaps

- Checkout is a UI-only flow; no payment integration.
- Only tested in modern Chromium/Firefox.
